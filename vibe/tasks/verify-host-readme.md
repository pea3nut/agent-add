# Task: Verify and Update src/hosts/README.md

## 目标

逐一访问 `src/hosts/README.md` 中每个宿主的官方文档链接，验证并补全每个资产类型的「文档 URL」和「文档原文摘录」字段。修复所有过时或错误的信息。

## 需要验证/更新的文件

- `src/hosts/README.md` — 唯一信源，本次任务的主要修改目标
- `src/hosts/<id>.ts` — 每个宿主的适配器文件，`docs` 等字段需与 README 同步
- `README.md` — 主文档（英文），含宿主相关信息需同步
- `docs/README.zh-CN.md` — 中文翻译文档，需同步
- `docs/README.ja.md` — 日文翻译文档，需同步

## README 当前格式

每个宿主的格式为：

```markdown
### Host Name

官网：https://...

| 资产类型 | 文档 URL | 文档原文摘录 | 关键信息记录 |
|----------|----------|-------------|-------------|
| MCP | ... | ... | ... |
| Prompt | ... | ... | ... |
| Skill | ... | ... | ... |
| Command | ... | ... | ... |
| Sub-agent | ... | ... | ... |
```

- **官网**：产品文档首页 URL
- **文档 URL**：该资产类型对应的官方文档页面链接
- **文档原文摘录**：从官方文档中摘录的关键原文（路径、key 名、格式等）
- **关键信息记录**：我们项目中使用的配置路径、strategy 等技术细节
- 不支持的资产类型，三列均填 ❌

### 每个宿主的验证流程

#### Step 1: 访问官网

- 访问该宿主的「官网」URL
- 确认链接可达（404 / 重定向均需更新）
- 浏览文档首页，寻找各资产类型的文档入口

#### Step 2: 逐行验证表格

对表格中的每一行（MCP / Prompt / Skill / Command / Sub-agent）：

| 检查项 | 说明 |
|--------|------|
| 文档 URL | 找到该资产类型的官方文档页面，填入具体 URL；找不到则填（未知） |
| 文档原文摘录 | 从文档页面中摘录关键原文（配置路径、key 名、文件格式等）；页面无相关内容则填（未知） |
| 关键信息记录 | 将摘录内容与现有记录对比，有差异则更新；无差异则保留 |

重点确认：
- MCP 配置文件路径（各平台）
- MCP 配置 key 名称（`mcpServers` / `servers` / `mcp` 等）
- Prompt 路径/策略（单文件 `append-with-marker` 还是目录 `create-file-in-dir`）
- 配置格式（JSON / TOML）

#### Step 3: 立即更新 README（增量式）

> **调研完一个宿主，立刻更新 `src/hosts/README.md` 中该宿主的表格。**
> 不要等所有宿主调研完再一口气更新——每完成一个就写入一个，确保进度可见、中途中断不丢失。

- 将验证结果写入 `src/hosts/README.md` 对应宿主的表格中
- 如发现「关键信息记录」有误，同步修正

#### Step 4: 报告该宿主结果

完成一个宿主后，简要报告：
- 更新了哪些字段
- 发现了哪些差异
- 是否有无法确认的信息

然后继续下一个宿主。
